## Overview

The Company page manages company-level configuration for payroll processing. It uses a **Tab layout** to organize different configuration areas.

**Key Changes from Previous Design**:
- Renamed from "Settings" to "Company"
- Added Tab navigation (Profile, Pay Groups, Integration)
- Added Pay Groups management section
- Removed standalone Payroll Settings section (moved to Pay Groups)

## Accessibility

- Tab navigation follows WAI-ARIA tab pattern (`role="tablist"`, `role="tab"`, `role="tabpanel"`)
- All form fields have associated labels via `<label for="...">` or `aria-labelledby`
- Required fields marked with asterisk (*) and `aria-required="true"`
- Error messages linked to fields via `aria-describedby`
- Toggle switches have `role="switch"` and `aria-checked`
- Focus visible on all interactive elements with outline styling
- Modals trap focus and return focus to trigger element on close
- Destructive buttons use distinct color (`--color-error-500`) for visual clarity
- Info boxes use `role="note"` for screen readers

## State Management

```typescript
// payroll-frontend/src/lib/stores/companyStore.ts

import { writable, derived } from 'svelte/store';
import type { CompanyProfile, BookkeepingSyncStatus } from '$lib/types/company';
import type { PayGroup } from '$lib/types/pay-group';

// Company profile state
export const companyProfile = writable<CompanyProfile | null>(null);
export const isLoading = writable(false);
export const error = writable<string | null>(null);

// Pay groups state
export const payGroups = writable<PayGroup[]>([]);

// Bookkeeping sync status
export const syncStatus = writable<BookkeepingSyncStatus | null>(null);

// Derived: is connected to bookkeeping
export const isConnectedToBookkeeping = derived(
  companyProfile,
  ($profile) => $profile?.bookkeepingLedgerId !== null
);

// Derived: total employees in all pay groups
export const totalEmployeesInPayGroups = derived(
  payGroups,
  ($groups) => $groups.reduce((sum, g) => sum + g.employeeCount, 0)
);
```
